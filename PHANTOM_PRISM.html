Case File #003: PHANTOM PRISM

III. INVESTIGATOR'S SUMMARY

The objective was to materialize digital constructs into physical space, bending light to create three-dimensional apparitions. We needed an interactive, optical laboratory capable of projecting volumetric models without the constraints of multi-million-dollar holographic hardware. I engineered a decoupled, dual-device network where a portable terminal (Phone A) serves as the light source for a 4-split pyramidal projection, while a central command station (the Laptop) remotely dictates the temporal flow of the illusion via a localized web bridge.

IV. EVIDENCE LOG (Tech Stack)

  • The Network Bridge: Python, Flask (Local server routing and API endpoints)
  • The Command Station: HTML/CSS/JavaScript (Client-side control dashboard on the laptop)
  • The Optical Projector: Smartphone Screen (Displaying HTML5/Video rendering of 4-split models)
  • The Medium: Physical Transparent Pyramid (Pepper's Ghost reflection matrix)

V. MODUS OPERANDI (How It Works)

The investigation utilizes a localized client-server architecture to alter physical optics in real-time:

  • The Local Web: The Python Flask application boots up on the laptop, establishing a local area network (LAN) server. This creates an invisible communication tether between the laptop and Phone A.
  • The Command Interface: The investigator operates a web dashboard on the laptop. Pressing "Play," "Pause," "Next," or "Back" dispatches asynchronous HTTP requests (or WebSocket signals) to the Flask server.
  • The Optical Cast: Phone A, connected to the same Flask server as a client, listens for these state changes. Upon receiving a command, its browser or application instantly updates the media state of the 3D model.
  • The Physical Manifestation: The phone displays the targeted 3D model replicated into four perfectly symmetrical angles (front, back, left, right).

These four light streams hit the angled faces of the physical pyramid resting on the screen, converging into a single, floating holographic illusion in the center of the prism.

VI. THE VERDICT

The system effectively bridges the gap between digital control and physical manifestation. It delivers a seamless, remote-operated optical illusion that brings 3D models into the physical environment. By hijacking the fundamental principles of light reflection and pairing it with a lightweight Flask backend, the project achieves a highly interactive "Pepper's Ghost" hologram without requiring specialized projection hardware.

VII. CASE ASSETS

  • Asynchronous Command Protocol: Utilizes Flask routes to process control inputs instantaneously, ensuring the holographic apparition reacts to laptop commands without latency or buffering.
  • Symmetrical Spatial Rendering: The models are explicitly mathematically formatted into a precise 4-way split, guaranteeing proportional and distortion-free reflection across all four quadrants of the pyramid.
  • Decoupled Architecture: By separating the controller (laptop) from the display unit (phone), the system allows the investigator to manipulate the entity from the shadows without physically obstructing the viewer's line of sight.
