Case File #001: SCHOLAR'S FATE

III. INVESTIGATOR'S SUMMARY

The objective was clear: eliminate the guesswork from student performance. We needed a system capable of foreseeing which subjects were on a trajectory toward academic failure and which would pass. I built a predictive web interface to crack their patterns, utilizing historical data to forecast future outcomes.

IV. EVIDENCE LOG (Tech Stack)

  • Core Engine: Python, Scikit-Learn (Logistic Regression model)
  • Data Processing: Pandas, NumPy
  • The Front (UI/Backend): Flask, HTML/CSS
  • Data Preservation: Joblib (Model and Scaler serialization)

V. MODUS OPERANDI (How It Works)

To get a prediction, the system demands three key pieces of intel on the suspect (student):

  • Study Hours: The amount of time invested off-the-grid.
  • Attendance: Their physical presence on the record.
  • Previous Marks: Their historical track record.

Once the intel is submitted via the web interface, the system processes it. The raw data is passed through a pre-trained StandardScaler to normalize the inputs, ensuring no single variable skews the investigation. Finally, it's fed into a Logistic Regression model trained on historical cases.

VI. THE VERDICT

The application immediately returns a definitive ruling: PASS or FAIL, allowing for preemptive intervention before the academic term concludes.

VII. CASE ASSETS

  • Training Protocol: Utilized an 80/20 train-test split on the initial students.csv database to ensure the model's accuracy on unseen subjects.
  • Seamless Interface: A lightweight, gradient-styled HTML/CSS dashboard designed for rapid data entry and instant processing.
